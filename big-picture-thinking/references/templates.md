# Output Templates

Choose one template. Keep headings only when they improve clarity.

## One-Screen Big Picture

Use for quick answers.

```markdown
**Big Picture:** [One paragraph.]

**The 3 Drivers**
- [Driver 1]
- [Driver 2]
- [Driver 3]

**Tradeoff:** [The real choice.]

**Next Move:** [One practical action.]
```

## Deep-Dive Map

Use for complex or ambiguous problems.

```markdown
**Surface Problem**
[What the user is seeing.]

**Larger System**
[Where this problem comes from.]

**Main Drivers**
[The few causes that matter.]

**Incentives And Constraints**
[Actors, rewards, limits, and bottlenecks.]

**Tradeoffs**
[Real choices and costs.]

**Paths Forward**
1. [Path A: when it works, cost, risk]
2. [Path B: when it works, cost, risk]
3. [Path C: when it works, cost, risk]

**Recommended Next Move**
[Specific action.]

**What Would Change This**
[Key unknowns or assumptions.]
```

## Decision Memo

Use when the user needs to choose.

```markdown
**Decision**
[The choice being made.]

**Objective**
[What success means.]

**Constraints**
[What cannot be ignored.]

**Options**
- **A:** [best when, downside]
- **B:** [best when, downside]
- **C:** [best when, downside]

**Recommendation**
[Default choice and why.]

**Risk Control**
[How to reduce downside.]

**Next Step**
[Smallest action to validate or commit.]
```

## Systems Map

Use when incentives, feedback loops, or actors matter.

```markdown
**Actors**
[Who matters.]

**Flows**
[Money, attention, authority, work, information, trust.]

**Feedback Loops**
[What reinforces or dampens the pattern.]

**Bottleneck**
[The limiting factor.]

**Leverage Point**
[Where a small change matters.]

**Likely Side Effects**
[Second-order consequences.]
```

## Technical Zoom-Out

Use for engineering, architecture, debugging, or implementation strategy.

```markdown
**System Boundary**
[What is inside and outside the problem.]

**Current Failure Mode**
[What is actually breaking or constraining progress.]

**Root Cause Candidate**
[Most likely mechanism, with confidence.]

**Design Tradeoff**
[Correctness, complexity, performance, maintainability, security, migration, operability.]

**Better Boundary**
[Where responsibility should live.]

**Next Inspection Or Change**
[Concrete step.]
```

## Executive Brief

Use for concise strategic communication.

```markdown
**Bottom Line**
[Main conclusion.]

**Why It Matters**
[Strategic consequence.]

**Key Evidence**
[Only the strongest points.]

**Decision Needed**
[What must be decided.]

**Recommendation**
[What to do.]
```

## Questions-First

Use only when a useful answer is blocked by missing context.

```markdown
I can give a useful big-picture read, but one detail changes the answer:

[Question]

Why it matters: [How this changes the recommendation.]
```
